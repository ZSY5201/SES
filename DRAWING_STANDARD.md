# DRAWING_STANDARD

## 适用范围

本规范适用于 SES 中的系统图、控制框图、通信拓扑图、IO 图、流程图和调试示意图。

## 图纸目标

图纸应帮助快速理解系统结构、数据流、控制流和故障边界。

一张合格工程图应能回答：

- 系统由哪些部分组成？
- 信号从哪里来，到哪里去？
- 控制命令如何产生？
- 通信链路如何连接？
- 故障和安全边界在哪里？

## 通用要求

- 图名清晰。
- 版本明确。
- 日期明确。
- 方向一致，避免交叉线过多。
- 信号名称与代码、文档保持一致。
- 必要时标注单位、周期、协议和接口类型。

## 系统图

系统图应体现：

- 上位机
- 控制器
- 传感器
- 执行器
- 电源
- 通信链路
- 人机接口
- 数据记录或日志系统

## 控制框图

控制框图应体现：

- 目标值
- 当前值
- 误差
- 控制器
- 限幅
- 执行器
- 被控对象
- 反馈路径
- 故障或手动接管路径

## IO 图

IO 图应包含：

- 设备名称
- 信号名称
- IO 类型
- 地址
- 单位
- 量程
- 默认值
- 故障状态
- 备注

## 通信拓扑图

通信拓扑图应包含：

- 设备节点
- IP 地址或站号
- 协议
- 端口
- 通信方向
- 数据周期
- 超时设置
- 断线处理策略

## 文件命名

推荐命名方式：

```text
YYYY-MM-DD_系统或模块_图纸类型_v版本号.ext
```

示例：

```text
2026-07-05_USV控制系统_通信拓扑_v0.1.drawio
```
